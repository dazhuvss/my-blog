你好，我是 Yang。

这是我的个人技术博客，用来记录学习过程、整理项目经验，也把那些真正值得复盘的问题慢慢写清楚。

我希望这里不是一个只展示结果的站点，而更像一份长期更新的个人技术档案。它会保留我的学习轨迹、思考方式和那些真正踩过之后才理解的问题。

## 我在关注什么

目前我比较关注这些方向：

- Java 后端基础与工程实践
- Redis、MySQL、消息队列等常见后端组件
- AI Agent、工具调用与 RAG 相关实践
- 算法刷题、知识整理和面试复盘

我不太想把这个博客写成“只放结论的展示页”，更希望它像一个持续更新的技术笔记库。这里既会有系统化整理，也会有一些真实的学习过程、排查记录和阶段性复盘。

## 我为什么写这个博客

对我来说，写博客不是为了把内容堆得很多，而是为了把知识真正沉淀下来。

很多东西在“看懂”的时候，其实并不一定真的掌握了。只有把它重新组织成自己的语言，才能发现哪些地方还理解得不够，哪些问题值得继续往下挖。

所以这个博客会长期围绕三类内容展开：

- 学习笔记：记录我如何理解一个知识点
- 技术实践：整理真实问题的排查与解决过程
- 项目复盘：总结做项目时的设计思路和经验教训

## 你会在这里看到什么

如果你也在关注这些内容，可能会看到我写：

- 某个后端问题是怎么定位和修复的
- 某个知识点我是怎么从模糊到理解的
- 某个项目方案为什么这样设计
- 某段学习过程里值得保留下来的方法和思考
